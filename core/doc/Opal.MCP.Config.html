    Opal.MCP.Config (Opal v0.1.0)

    Copy Markdown View Source

    Discovers and parses MCP server configuration files.

    Searches multiple standard locations for mcp.json files following the VS Code MCP configuration format, and converts them into the internal %{name, transport} maps that Opal.MCP.Supervisor expects.

    Discovery paths (in order)

    Project-local (relative to working_dir):

    1. .vscode/mcp.json
    2. .github/mcp.json
    3. .opal/mcp.json
    4. .mcp.json

    User global:

    1. ~/.opal/mcp.json

    First definition wins per server name — project-local overrides global.

    VS Code format

    {
  "servers": {
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    },
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp"
    }
  }
}

    Stdio servers use command + args + optional env/envFile. HTTP/SSE servers use type ("http" or "sse") + url + optional headers.
